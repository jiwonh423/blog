stale-while-revalidate는 '캐싱 전략'의 일종이다.
 
데이터가 클라이언트에 한 번 전해진 이상, 클라이언트는 해당 데이터가 최신 데이터인지 더 이상 알 수가 없다.
 
EX) 주문을 할 때 나에게 보이는 화면은 “접수 전"인데 가게는 이미 주문 접수 상태인 상황
실제 server에서 response가 와서 주문 상태를 업데이트 시켜주기 전까지는 client에서는 이러한 stale 데이터가 존재하기 마련.
 
개발자가 컨텐츠를 로딩할 때 즉시성과 최신성간의 균형을 유지하는데 stale-while-revalidate 적용가능
Cache-Control: max-age=1, stale-while-revalidate=59
 
 
react-query와 swr는 swr 전략을 취한 대표적인 데이터 패칭 도구로 진행중인 프로젝트에 적용하면 아래와 같은 장점을 얻을 수 있다.
  • 사용자가 특정 지도의 내용을 수정하면 이 변경 사항이 다른 사용자에게도 실시간으로 표시되어 일관된 데이터를 유지 가능
